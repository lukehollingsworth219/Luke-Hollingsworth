DriveSenseTrainer — Real-Time AI Driving Coach

Built solo at my first hackathon (TreeHacks 2026). A standalone embedded device that detects harsh driving behavior in real time, then turns sensor logs into coaching feedback using the OpenAI API.

Skills learned:
Embedded systems IMU calibration + filtering Real-time event detection State machines / cooldown logic Node.js bridge + JSON OpenAI API integration

Overview

DriveSenseTrainer is a real-time AI driving coach that blends embedded sensing, signal processing, and modern AI into one cohesive product. An ESP32 + ADXL345 measures forward acceleration, braking, turning, and jerk events as you drive.

On-device logic handles bias calibration, noise filtering, event cooldowns, and a weighted scoring system to detect harsh behaviors without constant false positives. Feedback is immediate via a 16×2 LCD and buzzer.

After a session, the device exports structured JSON over serial to a Node.js bridge. From that log, the OpenAI API generates a concise natural-language coaching summary (and optionally can be routed into an AI avatar UI).

Why it mattered: it forced me to solve real-world engineering issues—calibration, timing, noise, integration, and end-to-end orchestration, all in a short timely manner.

How it works

Sensing: ESP32 reads ADXL345 over I2C at a fixed sample rate.

Signal handling: bias calibration + smoothing to reduce drift/noise.

Detection: thresholds + cooldown logic + weighted scoring to flag harsh accel/braking/turning/jerk.

Feedback: LCD + buzzer for immediate “coach” cues.

Session output: structured JSON log → Node.js bridge → AI summary.

Tech stack

  • Hardware: ESP32, ADXL345, 16×2 LCD, buzzer
  • Firmware: Embedded C++ (I2C, sampling loop, state machine)
  • Bridge: Node.js (serial → JSON)
  • AI: OpenAI API (coaching summary generation)
  • Optional: WebSockets / avatar UI for “human-like” feedback
